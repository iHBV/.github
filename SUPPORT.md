# Support

## Getting Help

For general questions, usage issues, or project-related support, please use the appropriate repository issue tracker when available.

When opening a support request, include enough detail to help us understand the problem quickly:

- Project or repository name
- What you are trying to do
- Expected behavior
- Actual behavior
- Relevant logs, screenshots, or output
- Environment details, if applicable

## Security Issues

Do **not** use public issues for suspected vulnerabilities or sensitive security concerns.

Please report security issues privately by emailing:

**honeybadger@ihbv.io**

See `SECURITY.md` for the full vulnerability disclosure process.

## Scope of Support

Support is provided on a best-effort basis for public repositories maintained under the iHBV organization. Response times may vary depending on project activity, complexity, and current maintenance priorities.

## Responsible Use

Please keep requests focused, reproducible, and relevant to the repository in question. Clear reports help us review issues faster and keep projects maintainable.
